CustomHud v4 Changelog

New Config Screen

  • Unlimited Profiles
  • Each profile has a name, keybind to switch to it, and the ability to bind its toggles
  • Ability to reorder profiles
  • Decide which profiles are used when using the cycle profiles keybind
  • Buttons to go to the docs and discord

New Language Features

  • Added List Variable Support
    • {<list>} alone will just show how many there are
      • Example: {effects} → 3 (assuming the player has 3 status effects)
    • Separators all you to set text to be put between entries, but not at the end
    • Filters allow you to filter the entries before looping
    • Inline Syntax
      • Syntax: {<list>, "stuff here"}
      • Example: {effects, "{index} - {icon} {name}\n"}
      • With Filter: {effects, "{e:name}", e:level > 2}
      • With Separator: {effects, "{e:name}", ", "}
      • With Both: {effects, "{e:name}", ", ", e:level > 2}
    • Multiline Syntax
      • Syntax: =for:<list>= ends with =endfor=
      • With Filter: =for: effects, e:level > 2=
      • Separator: everything after the =separator= will be the separator
      • Example: {effects, "{index} - {icon} {name}\n"}
    • Each list will have a set of child variables that only work within the list
      • Available to all lists:
        • {i} / {index} , {raw} , {size}
        • {exit} (exits the list prematurely), {continue} (skips to the next iteration)
      • The e: and likewise can be changed with the -pre:<prefix> list flag.
        • Ex: {effects -pre:eff, "{eff:name}"}
  • Added a lot of operators to conditionals (now called expressions)
    • Equals (= / ==) are no longer case-sensitive
    • Math Operators: + , - , * , / , %(modulus), and ^(exponential)
      • Ex: {{x > (8+2 *2, "X is greater than 10"}}
    • Ternary Operator: condition ? valueA ; valueB
      • Ex: {setsleep_time, raining ? 12542 ; 12010}
    • Not (!) Operator
      • Ex: {{ !(raining), "not raining","raining"}}
    • has Operator
      • Checks to see if a string has a string in it
        • Ex: {{item:main:name has "wool", "This has wool in the name"}}
      • Or checks to see if a list contains an entry
        • Example: {{items has "Stone", "I have Stone"}}
        • Can use child var to test for a specific value
          • <list>.<child> has <value>
          • Ex: {{all_items.count has 64, "I have at least 1 full stack"}}
    • Functions (all trig uses degrees)
      • Ex: {set:test, sin(30+60)}
      • Functions added:
        • round , ceil , floor
        • sqrt , abs
        • sin , cos , tan , csc , sec , cot
        • asin , acos , atan , acsc , asec , acot
        • log / log10 , log2 , log_e / ln , log_e1p / ln1p
    • If no value (??) Operator
      • Sometimes a variable doesn't have a usable number value. For example, {target_block_x} when not looking at a block. It can't use 0 or -1 as those are valid coordinates. So it instead gives the value of NaN (Not a Number).
      • When you do math with NaN value, the result will always be NaN
      • The ?? allows you to give a "default" value
      • Example:
        • Sharpness: {$ item:main:enchant:sharpness:lvl + 10}
        • If the item doesn't have sharpness it'll show as "Sharpness: -", but what if we wanted to treat not having the enchantment as having it at level 0
        • Sharpness: {$ (item:main:enchant:sharpness:lvl ?? 0) + 10}
    • Math symbol aliases for operators. For the lols, not really useful
      • And: ⋀ , ∧
      • Or: ⋁ , ∨
      • Sqrt: √(<expression>)
      • Has:∋ , ∍ , ∌ (list ∋ element), and ∈ , ∊ , ∉ (element ∈ list)
  • Expanded Profile Section with max Width and Text Align
    • Width
      • Width can now be set to max or -2. The background for every line will be as wide as the longest line
      • The default behavior can now be written as fit or -1
    • Text Align
      • ==Section:TopLeft,0,0,false,fit,<align>==
      • left , right , center
  • Ability to disable vanilla hud elements (like hotbar or effects)
    • ==Disable: <VanillaHudElements>==
    • Elements:
      • health , hunger , armor , air, (status_bars for all 4 + horse_health)
      • hotbar, item_tooltip , xp , (lower for both and everything in status_bars)

      • titles , actionbar
  • Ability to set the color based on the number value of a variable
    • Putting & in-front of any variable will now use its number value for color
      • Ex: &{target_block_color}
      • Various variables have been added to take advantage of this
      • Word colors like {red} are now proper variables and can be used in expressions
      • Can use expressions, Ex: &{target_block_color * -1}

New Variable Flags

  • -r<degrees> / -rotate<degrees>: Rotates Icons
    • Ex: {item:main:icon -r45} will rotate the icon 45 degrees
  • -zf# / -zerofill#: Pad the left with zeros when needed
    • Ex: {x -zf3} (5 would be 005)
  • -v# / -freq# / -frequency#
    • Sets how often the variable updates in seconds. Doesn't work on lists
    • Ex: -v2 , -v0.5 , -v1/20
  • For Identifier Variables
    • -ns / -namespace: Only shows the namespace
    • -path: Only shows the path
  • -sub / -subscript: Numbers are replaced with their subscript counterpart
  • -sup / -superscript: Numbers are replaced with their subscript counterpart
  • Flags that changes the number's base
    • -hex: Numbers are shown in hexadecimal
    • -oct: Numbers are shown in octal
    • -bin: Numbers are shown in binary
    • -base#: Numbers are show in the base specified.
      • Ex: {x -base7}
  • -rn / -roman: Displays the number in Roman Numerals
     

New Theming

  Scaling (Global only)
  • ==NiceScale: <(+/-) GUI Scale>==
    • Sets the scale of the CustomHud hud to as if it was a specific gui scale.
      This avoids weirdness that can happen when using ==Scale: <scale>==
    • ==NiceScale: 2== will render as if the GUI Scale in Minecraft's settings was set to 2.
    • ==NiceScale: +1== with your GUI Scale set to 3, the profile will render as if it was set to 4.
  • ==HudScale: <(+/-) GUI Scale>==
    • Same as NiceScale, but for minecraft's vanilla hud (hotbar, scoreboard, bossbar, etc)
  Visual (Works Globally and Locally)
  • ==Padding: <top>, <bottom>, <left>, <right>==
    • Default: 2,1,2,2
    • Can also be written as ==Padding: <verticle>, <horizontal>== or ==Padding: <all>==
    • Sets how much padding is around the text. 0 all around will be just big enough to fit the text and its shadow
  • ==PersistentFormatting: <true|false>==
    • Default: false
    • If true, color codes (like &a or &{green}) won't reset formatting (like bold)
  • ==ItemIconSize: <line|full>==
    • Default: line
    • If line, item icons will be scaled to fit the line (like all other icons). If full, it'll render at full size
  • ==LeftGraph: <graph>== and ==RightGraph: <graph>==
    • Default: None
    • Graphs: Profiler, Fps, Tick, Ping, PacketSize, None
  • ==WhenHudHidden: <Show|ShowIfScreen|Hide>== (global only)
    • Default: ShowIfScreen
    • Determines if the profile should show ehn the hud is hidden via F1
    • You can use {hud_hidden} in a conditional to limit what info is shown when the hud is hidden
  Change how the profile is read (Works Globally and Locally)
  • ==IgnoreBlankLines: <true|false>==
    • Default: false
    • Blank lines in the profile file are ignored. If true and you want to add blank lines, use \n
  • ==IgnoreBlankLines: <true|false>==
    • Default: false
    • Blank lines in the profile file are ignored. If true and you want to add blank lines, use \n
  • ==IgnoreLeadingSpace: <true|false>==
    • Default: false
    • Removes leading spaces on lines. This is useful for nested if/fors
  • ==ConvertLineBreaks: <true|false>==
    • If true it treats line breaks in the profile file as NewLines in the hud
    • If false, only \n in the profiles will add NewLines to the hud
    • This means that you can also change theming part way through a line.
    • Quirks when using =BackgroundColor:<color>= part way through a line
      • If the section's width is set to fit (default), it won't apply until after the next NewLine
      • If the section's width is set to max or a specific amount, it'll apply to the entire line
    • In a future release, there will be a local version of this
    • Default: true (same behavior as before)
